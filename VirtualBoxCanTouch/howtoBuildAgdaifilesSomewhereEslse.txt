How to Build .agdai Files Somewhere Else (Read-Only Cubical Library on a VM)
=============================================================================

PROBLEM
-------
The Cubical Agda library is mounted read-only (e.g. a VirtualBox shared
folder at /Cubical/).  Agda needs to write .agdai interface files into
the library's _build/ directory, but it can't because the filesystem is
read-only.

SOLUTION: bind mount
--------------------
Use a Linux bind mount to overlay a writable directory on top of the
read-only _build/ directory.  This makes Agda see the writable copy
when it tries to read or write _build/, while the rest of the library
remains read-only.

STEP-BY-STEP
------------

1. Create a writable directory for the build cache:

     mkdir -p /home/$USER/CubicalWritable/_build

2. (Optional) Copy existing .agdai files so you don't have to rebuild
   everything from scratch:

     cp -a /Cubical/_build/* /home/$USER/CubicalWritable/_build/

   This can take a moment (the Cubical library cache is ~400 MB).

3. Bind mount the writable directory over the read-only one:

     sudo mount --bind /home/$USER/CubicalWritable/_build /Cubical/_build

4. Verify it works:

     touch /Cubical/_build/testwrite && rm /Cubical/_build/testwrite && echo OK

   Should print "OK".

5. Now Agda can compile files that depend on the Cubical library:

     agda MyFile.agda    # will write .agdai files into the overlay

IMPORTANT NOTES
---------------

- The bind mount does NOT survive a reboot.  You must re-run the mount
  command (step 3) after every VM restart.

- To make it persistent across reboots, add this line to /etc/fstab:

    /home/<user>/CubicalWritable/_build  /Cubical/_build  none  bind  0  0

  Then run:  sudo mount -a

- The bind mount only shadows /Cubical/_build.  The Agda source files
  under /Cubical/Cubical/ remain read-only, which is what you want.

- If you upgrade the Cubical library or change Agda versions, you may
  need to clear the writable _build/ and let Agda rebuild:

     rm -rf /home/$USER/CubicalWritable/_build/*

- Make sure your ~/.agda/libraries file contains a line pointing to the
  Cubical library's .agda-lib file, e.g.:

     /Cubical/cubical.agda-lib

  And your project's .agda-lib should have:

     depend: cubical

ALTERNATIVE: symlink approach (does NOT work here)
--------------------------------------------------
You might think of symlinking /Cubical/_build -> writable location, but
this fails when /Cubical/ itself is read-only (you can't create or
replace the _build symlink).  The bind mount approach avoids this.

ALTERNATIVE: Agda --local-interfaces flag
-----------------------------------------
Agda has a --local-interfaces flag that stores .agdai files alongside
the source in a local _build directory (relative to the project, not the
library).  However, this only applies to the project being compiled, not
to library dependencies.  The library's own interface files still go
into the library's _build/.  So this does NOT solve the read-only
library problem.

SUMMARY
-------
The key command is:

  sudo mount --bind /home/$USER/CubicalWritable/_build /Cubical/_build

Run it once per boot (or add to /etc/fstab for persistence).
